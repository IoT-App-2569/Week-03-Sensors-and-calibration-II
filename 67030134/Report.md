## **ส่วนที่ 7 — การทดสอบผลลัพธ์**

ให้นักศึกษาทดสอบ 3 จุดแสงใหม่ที่ไม่ได้ใช้ตอนเก็บข้อมูล เช่น:

| สภาพแสงใหม่ | ADC  | Lux มือถือ | Lux จากโค้ด |
| ----------- | ---- | ---------- | ----------- |
| ใต้โต๊ะ     | 243  | 20         | 23          |
| ห้องเรียน   | 2515 | 130        | 694         |
| หน้าต่าง    | 3729 | 521        | 894         |

ให้นักศึกษาวิเคราะห์ว่า

- ความคลาดเคลื่อนอยู่ในระดับที่ยอมรับได้หรือไม่
-ยอมรับได้ครั แม้จะมีค่าคลาดเคลื่อนอยู่บ้าง แต่ถือว่าปกติเพราะเกิดจากมุมวัดที่ต่างกันระหว่างมือถือกับ LDR และสัญญาณรบกวน (Noise) 
ในขณะอ่านค่า ซึ่งไม่ส่งผลกระทบต่อการใช้งานจริง
- สมการ Linear หรือ Polynomial เหมาะกว่ากัน
-Polynomial (ดีกรี 2) เหมาะกว่าครับ **เหตุผล:** เนื่องจาก LDR เปลี่ยนความต้านทานแบบไม่เป็นเส้นตรง (Non-linear) การใช้ Polynomial จึงเก็บรายละเอียดความโค้งของกราฟได้แม่นยำกว่าการใช้ Linear ที่ค่าจะเพี้ยนมากในแต่ละช่วงแสง

## **ส่วนที่ 8 — คำถามท้ายใบงาน**

1. ทำไมต้องใช้ Voltage Divider ในการอ่าน LDR?
    ตอบ เพราะ LDR เปลี่ยนแค่ความต้านทาน ไม่ได้จ่ายแรงดัน แต่ ADC ของ ESP32 ต้องการแรงดันไฟฟ้า (Voltage) เพื่ออ่านค่า
2. ทำไมต้องเก็บข้อมูลหลายจุดก่อนทำ Calibration?
    ตอบ เพราะ LDR มีกราฟความสัมพันธ์แบบ Non-linear (Logarithmic) การเก็บหลายจุดทำให้สร้างสมการ Curve Fitting ที่แม่นยำกว่าการเก็บแค่ 2 จุด
3. ถ้า Noise มากกว่าสัญญาณจริง จะเกิดอะไรขึ้น?
    ตอบ **ถ้า Noise > สัญญาณจริง?** ค่าที่อ่านได้จะแกว่งมาก (Jitter) จนไม่สามารถแยกแยะความสว่างที่แท้จริงได้
4. ทำไมการวางตำแหน่งเซนเซอร์จึงสำคัญต่อ Signal Integrity?
    ตอบ เพื่อรักษา **Signal Integrity** ป้องกันเงาบังหรือแสงรบกวนจากทิศทางอื่น
5. ถ้า ADC มี Resolution ต่ำ จะส่งผลอย่างไรต่อความแม่นยำ?
    ตอบ จะทำให้ความละเอียดของค่าแสงน้อยลง เห็นการเปลี่ยนแปลงเป็นขั้นบันได (Quantization Error)
