# **ส่วนที่ 1 — ความรู้พื้นฐาน**

## **1.1 LDR คืออะไร**

![](Images/What_is_LDR.png)


LDR (Light Dependent Resistor) หรือ Photoresistor คืออุปกรณ์อิเล็กทรอนิกส์ชนิดหนึ่งที่มี **ค่าความต้านทานเปลี่ยนตามความเข้มของแสง** ที่ตกกระทบ

- **แสงมาก → ความต้านทานลดลง**
- **แสงน้อย → ความต้านทานเพิ่มขึ้น**

LDR จึงเหมาะสำหรับงานตรวจจับความสว่าง เช่น ระบบเปิดไฟอัตโนมัติ, ระบบวัดแสง, ระบบควบคุมหน้าจอ, ระบบความปลอดภัย ฯลฯ


## **1.2 โครงสร้างและหลักการทำงานภายใน**

LDR ทำจากสารกึ่งตัวนำ เช่น **Cadmium Sulfide (CdS)** หรือ **Cadmium Selenide (CdSe)**
### เมื่อมีแสงตกกระทบ:
1. โฟตอนของแสงกระตุ้นอิเล็กตรอนในสารกึ่งตัวนำ
2. อิเล็กตรอนหลุดจากแถบวาเลนซ์ไปยังแถบคอนดักชัน
3. จำนวนผู้พาไฟฟ้าเพิ่มขึ้น
4. ความต้านทานลดลง

### เมื่อไม่มีแสง:

- อิเล็กตรอนกลับสู่สภาวะปกติ
- ผู้พาไฟฟ้าลดลง
- ความต้านทานเพิ่มขึ้นมาก (อาจถึงหลายร้อย kΩ)

##  **1.3 พฤติกรรมเชิงคณิตศาสตร์ (Transfer Function)**

ความสัมพันธ์ระหว่าง **ความต้านทานของ LDR (R_LDR)** กับ **ความเข้มแสง (Lux)** ไม่เป็นเส้นตรง แต่เป็นแบบ **Exponential / Power Law**

### **สมการทั่วไปของ LDR**


$$R_{LDR}=A⋅(Lux)^{−α}
$$

โดย
	
	$A$ = ค่าคงที่ขึ้นกับรุ่นของ LDR

	$α$ = ค่าคงที่ (ประมาณ 0.7–0.9)
	
	$Lux$ = ความเข้มแสง

ความหมายคือ:
- เมื่อ Lux เพิ่มขึ้น → R ลดลงแบบโค้ง
- ไม่ใช่ความสัมพันธ์เชิงเส้น

เพื่ออ่านค่า LDR ต้องใช้วงจร **Voltage Divider** เพื่อแปลงความต้านทาน → แรงดัน → ADC อ่านได้

##  **1.4 ทำไมต้องใช้ Voltage Divider**

LDR ให้ “ความต้านทาน” แต่ไมโครคอนโทรลเลอร์อ่าน “แรงดัน” ดังนั้นต้องใช้วงจร **Voltage Divider** เพื่อแปลง R → V

### วงจร Voltage Divider

![](Images/LDR_and_R10k.png)
 **สมการแรงดันที่อ่านได้**


$$V_{out}=V_{cc}⋅\frac{R_{fixed}}{R_{LDR}+R_{fixed}}
$$

เมื่อ:

- $R_{LDR}$ ลดลง (แสงมาก) → $V_{out}$ เพิ่มขึ้น

- $R_{LDR}$ เพิ่มขึ้น (แสงน้อย) → $V_{out}$ ลดลง

## **ส่วนที่ 3 — การอ่านค่า ADC**

ตัวอย่างโค้ด ESP32 (ADC 12-bit)

```c
#include <stdio.h>
#include "driver/adc.h"
#include "esp_adc_cal.h"
#include "esp_log.h"

#define ADC_CHANNEL     ADC1_CHANNEL_6   // GPIO34
#define ADC_ATTEN       ADC_ATTEN_DB_11  // รองรับ ~3.3V
#define ADC_WIDTH       ADC_WIDTH_BIT_12 // 0–4095

static esp_adc_cal_characteristics_t adc_chars;

void app_main(void)
{
    // 1) ตั้งค่า ADC
    adc1_config_width(ADC_WIDTH);
    adc1_config_channel_atten(ADC_CHANNEL, ADC_ATTEN);

    // 2) ทำ Calibration (ใช้ค่า Vref = 1100mV)
    esp_adc_cal_characterize(
        ADC_UNIT_1,
        ADC_ATTEN,
        ADC_WIDTH,
        1100,              // ค่า Vref ของ ESP32 โดยประมาณ
        &adc_chars
    );

    while (1) {
        // 3) อ่านค่า ADC raw
        int raw = adc1_get_raw(ADC_CHANNEL);
        // 4) แปลงเป็นแรงดัน (mV)
        uint32_t voltage = esp_adc_cal_raw_to_voltage(raw, &adc_chars);
        printf("ADC Raw = %d, Voltage = %d mV\n", raw, voltage);
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}
```

ค่าที่อ่านได้ (12 bit) จะอยู่ในช่วง **0–4095**

```c

```

```c
Lux = 0.0001 × ADC^2 + 0.3 × ADC – 40
```

## **ส่วนที่ 6 — การทำ Calibration ในโค้ด**

ตัวอย่างแบบ Linear Fit

```c
float lux = 0.42 * raw - 50;
```

ตัวอย่างแบบ Polynomial:

```c
float lux = 0.0001 * raw * raw + 0.3 * raw - 40;
```

**หมายเหตุ** 

Code ที่ใช้ สามารถ reuse code  จากการ calibrate ค่ามุมในการทดลองที่แล้วได้

**การวิเคราะห์**

จากตารางทดสอบ 3 จุดแสงใหม่ คำนวณ % ความคลาดเคลื่อนระหว่าง Lux มือถือ กับ Lux จากโค้ดได้ดังนี้

| สภาพแสงใหม่ | Lux มือถือ | Lux จากโค้ด | % ความคลาดเคลื่อน |

- **ความคลาดเคลื่อนอยู่ในระดับที่ยอมรับได้หรือไม่:** ความคลาดเคลื่อนทั้ง 3 จุดอยู่ในช่วง 2.78–5.00% (เฉลี่ยประมาณ 3.9%) ซึ่งถือว่า**อยู่ในระดับที่ยอมรับได้** สำหรับเซนเซอร์ประเภท LDR ที่โดยธรรมชาติมีความไม่เป็นเชิงเส้นและมีความคลาดเคลื่อนจากปัจจัยแวดล้อมอยู่แล้ว (เช่น มุมตกกระทบของแสง, ความแม่นยำของ Lux Meter ในมือถือเอง) ค่า R² ที่ได้ทั้งสองชุด (0.9347 และ 0.9289) ก็สูงกว่า 0.9 ทั้งคู่ ซึ่งบ่งชี้ว่าสมการ Calibration อธิบายความสัมพันธ์ระหว่าง ADC และ Lux ได้ดีในระดับที่น่าพอใจ แม้จะยังไม่ถึง 1.000 ก็ตาม

- **สมการ Linear หรือ Polynomial เหมาะกว่ากัน:** เนื่องจากค่า R² ของทั้งสองชุดข้อมูลใกล้เคียงกันมาก (ต่างกันเพียงประมาณ 0.006) และ % ความคลาดเคลื่อนในการทดสอบก็อยู่ในระดับต่ำและใกล้เคียงกันทุกจุด ไม่มีจุดใดที่คลาดเคลื่อนมากผิดปกติโดยเฉพาะที่ปลายช่วง (เช่นที่ค่าแสงต่ำมากหรือสูงมาก) แสดงว่าในช่วงข้อมูลที่เก็บมา สมการ **Linear ก็ให้ผลลัพธ์ที่แม่นยำเพียงพอแล้ว** ไม่จำเป็นต้องใช้ Polynomial ซึ่งมีความซับซ้อนมากกว่า อย่างไรก็ตาม เนื่องจากความสัมพันธ์ทางฟิสิกส์ของ LDR จริง ๆ เป็นแบบ Exponential/Power Law (ไม่ใช่เชิงเส้น) หากต้องการใช้งานในช่วงแสงที่กว้างขึ้นมาก (เช่น ต่ำกว่า 5 Lux หรือสูงกว่า 1200 Lux) สมการ Polynomial อาจให้ความแม่นยำที่ดีกว่า เพราะสามารถจับความโค้งของกราฟที่ปลายช่วงได้ดีกว่าเส้นตรง

## **ส่วนที่ 8 — คำถามท้ายใบงาน**

1. ทำไมต้องใช้ Voltage Divider ในการอ่าน LDR?
Answer เพราะ LDR เป็นอุปกรณ์ที่เปลี่ยนแปลง "ความต้านทาน" ตามความเข้มแสง แต่ขา ADC ของไมโครคอนโทรลเลอร์อ่านค่าได้เฉพาะ "แรงดัน" เท่านั้น จึงต้องต่อ LDR ร่วมกับตัวต้านทานคงที่เป็นวงจร Voltage Divide
r เพื่อแปลงการเปลี่ยนแปลงความต้านทาน (R) ให้กลายเป็นการเปลี่ยนแปลงแรงดัน (V) ตามสัดส่วน แล้ว ADC จึงจะสามารถอ่านค่าและนำไปประมวลผลต่อได้

2. ทำไมต้องเก็บข้อมูลหลายจุดก่อนทำ Calibration?
Answer เพราะความสัมพันธ์ระหว่าง ADC กับ Lux ไม่ได้เป็นเส้นตรงแบบง่าย ๆ (เป็นแบบ Exponential/Power Law) การใช้ข้อมูลเพียง 1-2 จุดจะไม่สามารถระบุรูปแบบความสัมพันธ์ที่แท้จริงได้ และอาจทำให้สมการ Cali
bration คลาดเคลื่อนไปตามจุดที่วัด การเก็บข้อมูลหลายจุดในสภาพแสงที่แตกต่างกันช่วยให้เห็นแนวโน้มของกราฟชัดเจนขึ้น เลือก Trendline ที่เหมาะสมได้ และได้สมการที่มีค่า R² สูง ครอบคลุมช่วงการใช้งานจริง

3. ถ้า Noise มากกว่าสัญญาณจริง จะเกิดอะไรขึ้น?
Answer ถ้า Noise มีขนาดใกล้เคียงหรือมากกว่าสัญญาณจริง (Signal-to-Noise Ratio ต่ำ) ค่า ADC ที่อ่านได้จะแกว่งไปมาอย่างไม่สม่ำเสมอ ไม่สะท้อนความเข้มแสงที่แท้จริง ทำให้สมการ Calibration ที่คำนวณจากข้อมู
ลนี้ผิดเพี้ยน และเมื่อนำไปใช้งานจริงค่า Lux ที่คำนวณได้จะไม่แม่นยำหรือกระโดดผิดปกติ ในกรณีรุนแรง สัญญาณจริงอาจถูกกลบจนไม่สามารถแยกแยะการเปลี่ยนแปลงของแสงออกจาก Noise ได้เลย

4. ทำไมการวางตำแหน่งเซนเซอร์จึงสำคัญต่อ Signal Integrity?
Answer ตำแหน่งของเซนเซอร์มีผลโดยตรงต่อคุณภาพสัญญาณที่วัดได้ เช่น การวางสายสัญญาณใกล้กับสายไฟกำลังหรืออุปกรณ์ที่มี Electromagnetic Interference (EMI) จะทำให้เกิด Noise แทรกเข้ามาในสัญญาณ (Crosst
alk) หรือหากเซนเซอร์วางในตำแหน่งที่รับแสงไม่สม่ำเสมอ (เช่น มีเงาบังบางส่วน หรือมีแสงสะท้อนแทรกซ้อน) ค่าที่วัดได้ก็จะไม่สะท้อนสภาพแสงจริงที่ต้องการวัด นอกจากนี้สายสัญญาณที่ยาวเกินไปหรือไม่มีการต่อกราวด์ที่ดีอาจทำ
ให้เกิด Ground Loop หรือ Voltage Drop ซึ่งลดทอนความถูกต้องของสัญญาณก่อนถึง ADC

5. ถ้า ADC มี Resolution ต่ำ จะส่งผลอย่างไรต่อความแม่นยำ?
Answer ADC ที่มี Resolution ต่ำ (จำนวนบิตน้อย) จะแบ่งช่วงแรงดันอินพุตออกเป็นระดับ (Step) ที่หยาบกว่า เช่น ADC 8 บิต แบ่งได้เพียง 256 ระดับ เทียบกับ 12 บิตที่แบ่งได้ 4096 ระดับ ทำให้ค่าที่อ่านได้มีความละเอีย
ดต่ำ ไม่สามารถแยกแยะการเปลี่ยนแปลงของแสงที่มีขนาดเล็กได้ ส่งผลให้กราฟ ADC vs Lux มีลักษณะเป็นขั้นบันได (Quantization Error) และสมการ Calibration ที่ได้จะมีความแม่นยำลดลง โดยเฉพาะในช่วงแสงที่มีกา
รเปลี่ยนแปลงน้อย ๆ ซึ่งจะถูกปัดเป็นค่าเดียวกันทั้งที่ความเข้มแสงจริงต่างกัน

## **ส่วนที่ 9 — ผลลัพธ์ที่คาดหวัง**

ผู้เรียนจะสามารถ:
- ต่อวงจร LDR ได้อย่างถูกต้อง
- อ่านค่า ADC ได้
- เก็บข้อมูลจริงและสร้างกราฟความสัมพันธ์
- ทำ Calibration เพื่อแปลง ADC → Lux
- ประเมินความแม่นยำของระบบวัดแสงที่สร้างขึ้นเอง

###  **คำถามสะท้อนความเข้าใจ (Reflection)**

ให้นักศึกษาเขียนตอบสั้น ๆ

1. จุดไหนในใบงานที่คิดว่ายากที่สุด? 
Answer ส่วนที่ยากที่สุดคือการต่อวงจรและตรวจสอบว่าจุดที่ต่อ GPIO34 อ่านค่าได้ถูกต้องจริง เพราะตอนแรกโค้ดรันได้ไม่มี error แต่ค่า ADC Raw กลับอ่านได้ 0 ตลอดเวลา ทำให้ต้องไล่ตรวจว่าปัญหาอยู่ที่โค้ดหรือฮาร์ดแวร์ สุดท้ายพบว่าเป็นปัญหาการต่อวงจร (จุดกึ่งกลางของ Voltage Divider ไม่ได้ต่อเข้า GPIO34 จริง) ไม่ใช่ปัญหาที่โค้ด ทำให้เข้าใจว่าการ debug เซนเซอร์ต้องแยกระหว่างปัญหาซอฟต์แวร์กับฮาร์ดแวร์ให้ออกจากกัน

2. ถ้าทำใบงานนี้อีกครั้ง จะปรับปรุงอะไร? 
Answer จะใช้มัลติมิเตอร์วัดแรงดันที่จุดกึ่งกลางของวงจร Voltage Divider ก่อนเริ่มเขียนโค้ด เพื่อยืนยันว่าวงจรถูกต้องตั้งแต่แรก จะได้ไม่เสียเวลาสงสัยว่าปัญหาอยู่ที่โค้ดหรือวงจร และจะเก็บข้อมูล ADC vs Lux จากหลายสภาพแสงให้ครบตามจำนวนก่อนเริ่มทำกราฟ เพื่อให้สมการ Calibration แม่นยำขึ้น

3. ส่วนไหนที่คิดว่าตนเองเข้าใจดีมาก? 
Answer เข้าใจดีมากในส่วนของหลักการเขียนโค้ดอ่านค่า ADC ด้วย ESP-IDF (adc1_config_width, adc1_config_channel_atten, esp_adc_cal_characterize, adc1_get_raw) และการนำสมการ Trendline (Linear/Polynomial) จากกราฟมาเขียนเป็นฟังก์ชันแปลงค่า ADC เป็น Lux ในโค้ด รวมถึงเข้าใจว่าทำไมต้องใช้วงจร Voltage Divider เพื่อแปลงความต้านทานของ LDR เป็นแรงดันที่ ADC อ่านได้
